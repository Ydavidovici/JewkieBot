Chess Engine: Opening Styles & Specific Openings Implementation Plan

Overview:
We will add the ability to force the engine to play specific styles (Positional, Tactical, Balanced) using your Polyglot (.bin) file, as well as force specific openings (e4, Sicilian, English, etc.) from the frontend.

Step 1: Setting up the Opening Books
------------------------------------
1. Find your Polyglot opening book (.bin file) on your home PC.
2. Copy the .bin file into your engine's build directory (e.g., `Chess-Engine/engines/myengine/build/books/`).
3. You can either rename it to `balanced.bin`, `tactical.bin`, etc., or keep its name and update the engine code below to point to its exact filename.

Step 2: Engine Changes (engines/myengine/src/main.cpp)
-------------------------------------------------------
1. In `handle_uci`, expose a new UCI Option for OpeningStyle:
   std::cout << "option name OpeningStyle type combo default Balanced var Balanced var Positional var Tactical\n";

2. In `handle_setoption`, add logic to handle `OpeningStyle` being sent by the backend. When received, map the style to the corresponding .bin file you placed in the build directory in Step 1, and load it into the engine using `engine.loadOpeningBook(path)`.
   Example:
   if (name == "OpeningStyle") {
       std::string bookPath = "books/" + value + ".bin"; // Make sure to lowercase 'value' if needed
       engine.loadOpeningBook(bookPath);
   }

Step 3: Backend Changes (backend/src/)
--------------------------------------
1. Create a new file `openings.js` that exports a dictionary mapping opening names to their initial move sequences:
   const specificOpenings = {
       "Starting Position": [],
       "e4": ["e2e4"],
       "d4": ["d2d4"],
       "English": ["c2c4"],
       "Sicilian": ["e2e4", "c7c5"],
       "Queen's Gambit": ["d2d4", "d7d5", "c2c4"],
       "London": ["d2d4", "d7d5", "c1f4"],
       "King's English": ["c2c4", "e7e5"],
       "Symmetrical English": ["c2c4", "c7c5"],
       "Najdorf Sicilian": ["e2e4", "c7c5", "g1f3", "d7d6", "d2d4", "c5d4", "f3d4", "g8f6", "b1c3", "a7a6"]
   };
   export default specificOpenings;

2. In `engineManager.js`, add a `setOption(name, value)` method to the `UciEngine` class that sends `setoption name <name> value <value>` to the engine via `_sendRaw()`.

3. In `server.js`, expose two new endpoints for the frontend to hit:
   - GET `/api/openings` -> returns the `specificOpenings` JSON.
   - POST `/api/engine/setoption` -> calls `manager.getEngine("Main").setOption(name, value)`.

Step 4: Frontend Changes (frontend/src/)
----------------------------------------
1. In `services/api.js`, add generic Axios wrappers to call `GET /api/openings` and `POST /api/engine/setoption`.

2. In `pages/Game.jsx`:
   - Fetch the openings list on mount (`useEffect`).
   - Add two new dropdowns: one for "Playing Style" (Balanced, Positional, Tactical) and one for "Specific Opening" (from the fetched list).
   - Update the `handleReset` function so that when a new game starts, it:
     a. Checks if a Specific Opening was selected. If so, apply those forced moves sequentially to the new `Chess()` instance using `newGame.move(...)`.
     b. Calls `setEngineOption("OpeningStyle", playingStyle)` to inform the backend of the style choice before the engine begins analyzing.
     c. If the forced moves result in it being Black's turn (and the engine plays Black), call `makeEngineMove(newGame.fen())` immediately so the engine responds to the opening sequence.
